Overview

Hex is a minimalist Godot 4.6 + C# turn-based piece-movement puzzle. A hexagonal grid (radius 4, 61 tiles) hosts a single token; the player selects one of 18 distinct piece types (Walker, Runner, Jumper, Knight, Stepper, …) and taps a tile to move there. Each piece encodes its own movement rule — single-step, sliding-ray, teleport, ring-locked, mirror, etc. There is no CLAUDE.md; the project is fully self-documenting via code.

Scene Flow

Single persistent scene. No transitions — token swaps happen via HexBoard.SetToken(index) when a UI button is pressed.

Scene tree (game.tscn):

  • Game (Node3D, GameScreen script)
    • WorldEnv — ambient 0.5 energy, sky RGB(0.08, 0.09, 0.13)
    • Camera3D — orthographic look-down at (-2, 6, 0), rotation 90°/0/0
    • Light — DirectionalLight3D, 1.1 energy, 45° angled overhead
    • HexBoard — Radius=4, builds 61 tiles on _Ready
    • UI (CanvasLayer) — Root Control + LeftPanel (640 px token picker) + Description label

Gameplay Loop

  1. Init: HexBoard._Ready builds tiles from HexCoord.Within(4), creates one Area3D + Mesh per tile, connects Area3D.InputEvent to a per-tile Callable. GameScreen._Ready iterates TokenCatalog.All to build 18 toggle buttons; pre-selects Walker (index 0).
  2. Selection: tap a tile → Area3D.InputEvent → OnTileInput(coord, event). If !_selected and coord == tokenPos, call BeginSelect() (memoised on (_lastSelectedToken, _lastSelectedPos) — returns early if same token/pos).
  3. Move generation: Token.LegalMoves(tokenPos, Radius, _movesBuffer) fills the buffer with destination HexCoords. Token.Filter strips origin tile + out-of-bounds in place.
  4. Highlight: for each move, find the tile in _tiles dict, swap Mesh.MaterialOverride to HighlightMaterial (emissive gold). Store highlighted coords in _highlighted HashSet.
  5. Commit: second tap on a highlighted tile → MoveTokenTo(coord). Set _tokenPos = coord, create Tween to animate the token over 0.18 s, Sine-out easing. EndSelect() restores base materials.

Each subclass returns a static-readonly shared Mesh + Material via GetSharedMesh() / GetSharedMaterial() — Cylinder, Sphere, Torus, Box, Capsule, or Prism in distinct colours, all with metallic 0.1, roughness 0.4, emission 0.25× base.

TokenCatalog is a TokenInfo record (Name, Description, Func<Token> Factory) — lets GameScreen build the picker UI without instantiating tokens.

Hex Math

Located in scripts/Hex/. Designed for zero-allocation use throughout the game.

HexCoord

  • Readonly struct (Q, R, S = -Q-R). Immutable.
  • 6 static Direction vectors: E, NE, NW, W, SW, SE.
  • Methods: +(HexCoord), -(HexCoord), *(int k), Distance(), DistanceFromOrigin(). Equality + hashing for O(1) dict lookup.
  • Enumerable forms: Within(radius) (yields all coords in circle), Ring(radius) (yields border only).
  • Buffer-filling overloads: Within(radius, List<HexCoord> output), Ring(radius, List<HexCoord> output) — used in hot loops to avoid allocations.

HexLayout

Static helper. TileSize = 0.55f. ToWorld(HexCoord h, float y = 0) uses flat-top hex math (√3 × 1.5 scale factors).

Board

HexBoard.cs:

  • Export Radius (default 4) — 61 tiles at default.
  • BuildBoard() pre-allocates all tiles on _Ready. Nested Tile class (Coord, CheckerIndex, Mesh, Area3D, InputHandler Callable).
  • Shared static CylinderMesh + CylinderShape3D for all 61 tiles — single allocation, GPU batching.
  • _movesBuffer List<HexCoord> (capacity 64) reused across all BeginSelect calls.
  • Memoises BeginSelect result (_lastSelectedToken, _lastSelectedPos, _selectionValid) — skip rebuild if same token/position tapped twice.
  • 3-colour checker via ((Q - R) % 3): TileMaterialA/B/C.
  • MoveTokenTo uses Tween to slide 0.18 s.

UI & Debug

  • GameScreen — Node3D root. Builds toggle button VBox from TokenCatalog.All; wires presses to OnPickToken(index). Listens to DebugLog.GameplayActiveChanged to gate Viewport.PhysicsObjectPicking.
  • DebugModal — full-screen dim backdrop + centred panel with RichTextLabel showing DebugLog.Snapshot(). Buttons: Refresh, Copy (flash toast), Clear, Close.
  • SettingsModal — 320 px right-edge drawer, slides in/out with tweens (0.22 s Cubic). Two buttons: Resume, Logs (opens DebugModal).

Performance Architecture

Audit health 8.5/10 per PERFORMANCE_AUDIT.md.

Already optimised

  • HexCoord readonly struct with proper hashing — O(1) dict lookup.
  • Within / Ring have buffer-filling overloads; code uses them everywhere.
  • All 18 Token subclasses expose static readonly shared Mesh + Material — GPU batching by piece type.
  • HexBoard uses shared static CylinderMesh + CylinderShape3D for all tiles.
  • _movesBuffer pre-allocated capacity 64, reused.
  • BeginSelect memoises (token, pos) and skips LegalMoves if unchanged.
  • Token.Filter uses in-place two-pointer compaction (no RemoveAll lambda).
  • MoveTokenTo creates one Tween per move — bounded by player tempo.

Minor issues

  1. Diagnostic GD.Print calls on every input event (HexBoard.cs lines 90, 98, 103, 197, 204, 210; GameScreen.cs line 79). Most are #if DEBUG-gated, line 79 is not. Remediation: move to DebugLog.Trace([Conditional("DEBUG")]).
  2. Per-tile Callable closure captures HexCoord (HexBoard.cs:183–184) — 61 closures at radius 4. Minor; convert to board-level pick handler with reverse Area3D→coord dict (Priority 3, optional).
  3. Viewport.PhysicsObjectPicking enabled by default; gated on/off by DebugLog.GameplayActiveChanged. Appropriate for touch UI.
  4. No shaders — all visuals via StandardMaterial3D, good for GL Compatibility.

Rendering

  • GL Compatibility renderer; 1280×720, canvas_items stretch, expand aspect, landscape orientation.
  • Tiles: CylinderMesh (6 radial segments, height 0.15 f, radius ≈ 0.52 f = TileSize × 0.95). 3-colour checkerboard. Highlight emits gold (0.95, 0.85, 0.30) at 0.6× intensity.
  • Tokens: Cylinder, Sphere, Torus, Box, Capsule, or Prism — distinct colour per piece. Metallic 0.1, roughness 0.4, emission 0.25× base.
  • Lighting: DirectionalLight3D (1.1 energy, 45° overhead). WorldEnvironment with ambient 0.5 energy, sky RGB(0.6, 0.65, 0.8).
  • Camera: orthographic, fixed look-down at (-2, 6, 0), 90° pitch.
  • No custom shaders.

Input

No custom input map. Default Godot bindings used.

  • InputEventScreenTouch — primary on mobile.
  • InputEventMouseButton (Left) — desktop tap.
  • Viewport.PhysicsObjectPicking enabled (default true) so Area3D.InputEvent signals fire.
  • HexBoard._Input logs diagnostic data per event (DEBUG-gated).
  • Area3D.InputEvent per tile routed to OnTileInput, which calls OnTileTapped on Left-click / touch-Pressed.
